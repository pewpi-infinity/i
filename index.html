Pewpi Encoder (in-page)

This built-in encoder encrypts or decrypts arbitrary text locally in your browser using a passphrase. It uses PBKDF2 → AES‑GCM. The page does NOT send your passphrase anywhere unless you explicitly paste the encoded output into a server field yourself. This is client-side only.
How it works (short): a 16‑byte random salt + 12‑byte random IV are generated. Key is derived with PBKDF2(SHA‑256) using the passphrase and salt. AES‑GCM encrypts data. The single base64 string contains salt+iv+ciphertext for safe storage.

Security notes (short): this runs in your browser. Anyone with your passphrase or the encoded blob and the passphrase can recover plaintext. The page cannot guarantee exclusivity — browser JavaScript can be inspected. Keep the passphrase secret and do not paste raw secrets into public inputs.
